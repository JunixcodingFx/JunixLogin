# Security Policy

## Supported Versions

Currently, JunixLogin is a simple frontend template and does not include backend authentication or sensitive data handling.

The latest version is actively maintained. Older versions are not officially supported.

| Version | Supported |
| ------- | --------- |
| 1.x     | Yes       |
| < 1.0   | No        |

***

## Reporting a Vulnerability

If you discover a security vulnerability in JunixLogin, please report it responsibly.

You can do this by opening a private issue or contacting the maintainer directly via GitHub.

When reporting a vulnerability, please include:

* A clear description of the issue
* Steps to reproduce the problem
* Possible impact or risk
* Suggested fix (if available)

All reports will be reviewed as quickly as possible.

If the vulnerability is confirmed, a fix will be released in a future update.  
If it is not considered a valid issue, you will receive an explanation.

***

## Notes

Since this project is a frontend template without backend logic, most security concerns will depend on how it is integrated into a larger system.
